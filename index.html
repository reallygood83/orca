조율 모드를 설계하세요

Orca Mode Studio는 누구나 자신의 multi-agent 워크플로를 정의하고, 플레이북·Quick Command·에이전트 스킬로 내보낼 수 있는 범용 설계 도구입니다. 팀장(coordinator) + 실무 풀(Codex / Claude / Grok / Gemini / Hermes 등)을 조합하세요. 설정은 서버에 저장되지 않으며, 사용자 기기 로컬에 설치됩니다.

Draft mode: MyMode Coordinator Grok · Workers — · Concurrency 3 · Path ~/.orca/my-mode/

제품이 해결하는 문제

AI CLI(Codex, Claude Code, Grok 등)는 강력하지만, 여러 에이전트를 동시에 쓰면 역할·모델·완료 신호·결과 합치기가 제각각입니다. Mode Studio는 Orca orchestration 엔진 위에 올릴 운영 모드(OS 레이어)를 설계하게 합니다.

설계 (이 사이트) 로컬 설치 Orca에서 실행
Supervised mode (기본)

Coordinator가 task를 만들고, worker에 dispatch --inject 한 뒤 worker_done까지 기다린 다음 FINAL을 작성합니다.

Handoff (선택)

다른 에이전트에 소유권을 넘기고 감시를 끊습니다. lifecycle inject를 쓰지 않습니다.

프라이버시 & 배포 모델

  • 1
    로컬 우선: 생성한 모드 설정은 각 사용자 기기(~/.orca/)에 저장됩니다.
  • 2
    서버 비저장: 이 사이트는 설정을 업로드하지 않는 정적 스튜디오입니다.
  • 3
    공유 방식: 스튜디오 URL 또는 export된 playbook/skill을 공유합니다.

실행 루프

Coordinator계획 · 배정
→
Workersmulti-model pool
→
FINAL합성 · 보고

동시 실행: 3 · Worktree: auto

예시 명령 (현재 드래프트 반영)

# Build a mode in tab 3 to preview commands
핵심: dispatch --inject 가 worker에게 완료 보고 규약(preamble)을 전달합니다. 모델은 dispatch가 아니라 각 worker의 --command에 지정합니다.
