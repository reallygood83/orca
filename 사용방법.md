# Orca Mode Studio — 사용 가이드

**제품 URL:** https://orca.teaboard.link  

이 문서는 **범용 제품**으로서 Mode Studio를 사용하는 방법을 설명합니다.  
특정 개인·채널 전용이 아닙니다.

---

## 이 제품은 무엇인가

| 구분 | 설명 |
|------|------|
| 하는 일 | multi-agent **조율 모드** 설계 · export · 로컬 설치 |
| 대상 | 개발자, 팀, 교육·커뮤니티 운영자 등 누구나 |
| 저장 위치 | 사용자 기기 (`~/.orca/`, `~/.agents/skills/`) |
| 서버 | 설정 업로드 없음 (정적 스튜디오) |

---

## 빠른 시작

### 1) 웹에서 모드 만들기
1. https://orca.teaboard.link 접속  
2. **Build mode** 탭  
3. Coordinator 선택  
4. Worker pool에서 Codex / Claude / Grok / Gemini / Hermes **복수 체크** 가능  
   - Hermes는 `hermes chat --tui` 권장. Orca `tui-idle` 미인식 시 수 초 대기 후 `dispatch --inject` 해도 worker_done 경로 확인됨.
5. Download pack 또는 Quick Command 복사  
6. Orca에 Quick Command 등록 후 Goal 실행  

### 2) 에이전트 스킬 설치
```bash
curl -fsSL https://orca.teaboard.link/setup-mode-pack.sh | bash
```
에이전트에게: `/orchestration-mode`

### 3) 엔진 스킬 (PC당 1회)
```bash
npx skills add https://github.com/stablyai/orca --skill orchestration
npx skills add https://github.com/stablyai/orca --skill orca-cli
```

---

## 핵심 개념

- **Coordinator**: 일 쪼개기, 배정, 대기, FINAL  
- **Workers**: 여러 명 가능 (모델·역할별 별도 터미널)  
- **Supervised**: `worker_done` 까지 기다림  
- **Handoff**: 소유권 이전, inject 감시 없음  

모델은 `dispatch` 옵션이 아니라 각 worker의 **실행 command**에 지정합니다.
